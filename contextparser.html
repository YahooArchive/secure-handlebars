Context-aware HTML5 Parser

Our novel approach to implement a HTML5 compliant parser for cross browsers.

How to solve XSS?

In order to mitigate the XSS vulnerability in the web applications, we need to address two biggest challenges.

Understanding the execution contexts according to the HTML5 WHATWG specification and apply context-sensitive encoding filters, unlike the widely-used Auto Escaping approach that blindly applies the same encoder on every output.
Addressing the cross browsers compatibility issue of some browsers do not fully compliant with the HTML5. This issue fails our execution contexts analysis and leads to wrong judgement of XSS filters.

Context Parser & HTML5 compliance

 

Context Parsing is our solution to understand the execution contexts of the HTML5 web pages according to the HTML5 WHATWG specification, and we mainly focus on the Tokenization process only, instead of implementing the end-to-end model of the HTML5 parser, to make our code base tiny and speedy.

Moreover, we address the cross browsers compatibility issue by a process called "canonicalization". In this process, we modify the inputting web page by rewriting the HTML to be HTML5 compliant, such that all browsers interpret the same for the page. With this strategy, all the downstream applications can focus on HTML5 specification without the concern of cross browsers compatibility issues.

Features of our Context Parser

 

HTML5 Compliance

We implement our HTML5 parser according to the HTML5 WHATWG specification. It is HTML5 Compliant!!!

Speedy

As our goal is to analyze the output execution context, there is no need to implement the end to end parsing model as the specification stated, so we can parse the HTML5 real fast.

Currently, we can parse 12.903225806451612 MB per second in our JavaScript implementation.

Tiny code base

Our code is around 500 lines, it is easy to be ported to different programming languages. Right now, we support native JavaScript implementation.

Canonicalization

It is a process to convert the inputting HTML and rewrite it to be HTML5 compliant.

Safe JavaScript Templating

Empowered by our Context Parser, we identify the HTML5 execution context before applying the corresponding filter on the output. This capability allows us to integrate with different JavaScript templating engines and make them XSS free.

What's WHATWG.org?

The Web Hypertext Application Technology Working Group (WHATWG) is a community of people interested in evolving HTML and related technologies.
